# Crushcode v0.2.3 Plan — TUI Polish & Markdown

## Goal
Make the TUI chat feel like a real AI coding assistant, not a prototype. Markdown rendering, status bar, and overall polish.

## Phases

### Phase 1: Markdown Rendering
**Why**: Assistant responses with code blocks, lists, bold are raw text right now. Unreadable.

- Parse markdown in assistant messages: headers, bold, italic, code blocks (```), inline code (`), lists (-, *), numbered lists
- Style each element with vaxis colors: headers=bright white bold, code blocks=cyan bg, inline code=cyan, bold=bold, italic=italic
- Code blocks: detect language tag, add border/box around code content
- Word-wrap with softwrap (already supported by vxfw.Text)
- Files: `src/tui/markdown.zig` (new)

### Phase 2: Status Bar
**Why**: No visibility into token usage, cost, session length.

- Bottom status bar (1 line): `Tokens: 1.2k in / 3.4k out | Cost: $0.05 | Turn 5 | 15m`
- Update after each AI response
- Dimmed style (ANSI 8) so it doesn't distract
- Files: edit `src/tui/chat_tui_app.zig`

### Phase 3: Header Polish
**Why**: Current header is minimal. Should show more useful info.

- Header: `Crushcode v0.2.3 | openrouter/claude-sonnet-4 | thinking:off | ctx: 45%`
- Context percentage: estimate from token count / model window
- Version in header
- Separator line under header
- Files: edit `src/tui/chat_tui_app.zig`

### Phase 4: Message Polish
**Why**: Messages need better visual hierarchy.

- User message: cyan left border `▌` (thicker), "You" label in cyan bold
- Assistant message: "Assistant" label in green bold, no border
- System/error messages: red style for errors, yellow for warnings
- Separator between turns: thin `─` line
- Timestamp on hover or compact mode
- Files: edit `src/tui/chat_tui.zig`, `src/tui/chat_tui_app.zig`

### Phase 5: Command Palette Polish
**Why**: Current palette works but looks basic.

- Add description column next to command name
- Show keyboard shortcut for each command
- Recent commands first
- Smooth open/close (stretch goal)
- Files: edit `src/tui/chat_tui_app.zig`

### Phase 6: Error Handling & Edge Cases
**Why**: Network errors, empty responses, rate limits should display gracefully.

- Network error: show red error message in chat, allow retry
- Rate limit: show "Rate limited, retry in Xs"
- Empty response: show "No response received"
- API key missing: show setup prompt instead of crash
- Config load failure: graceful fallback
- Files: edit `src/tui/chat_tui_app.zig`

## Not in v0.2.3 (deferred to v0.3.0)
- Tool call display
- Setup wizard TUI
- Install system (deb/rpm/brew)
- Syntax highlighting in code blocks
- File context indicator
- Slash commands in TUI (beyond palette)
