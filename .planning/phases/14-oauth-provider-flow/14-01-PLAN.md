**Key differences from mcp/oauth.zig:**
- mcp/oauth.zig uses standalone functions with `anytype` self parameter — this module uses a proper `ProviderOAuth` struct
- Token store path is `~/.crushcode/provider_tokens.json` (NOT `mcp_tokens.json`)

pub const ProviderOAuthConfig = struct {
    provider_name: []const u8,        // e.g. "openrouter"
    auth_url: []const u8,             // Authorization endpoint
    token_url: []const u8,            // Token exchange endpoint
    client_id: []const u8,            // OAuth client ID

Create `src/commands/auth_cmd.zig` — CLI handler for `crushcode auth <subcommand>`.

**Subcommands:**
1. `crushcode auth login <provider>` — Start OAuth browser flow
2. `crushcode auth status` — Show auth state for all providers
3. `crushcode auth logout <provider>` — Remove stored OAuth tokens

**handleAuth implementation:**

```zig
pub fn handleAuth(args: []const []const u8) !void {
    if (args.len == 0) {
        printAuthHelp();
        return;
    }
    const subcommand = args[0];
    if (std.mem.eql(u8, subcommand, "login")) {
        if (args.len < 2) { out("Usage: crushcode auth login <provider>\n", .{}); return; }
        try handleLogin(args[1]);
    } else if (std.mem.eql(u8, subcommand, "status")) {
        try handleStatus();
    } else if (std.mem.eql(u8, subcommand, "logout")) {
        if (args.len < 2) { out("Usage: crushcode auth logout <provider>\n", .{}); return; }
        try handleLogout(args[1]);
    } else {
        out("Unknown auth subcommand: {s}\n", .{subcommand});
        printAuthHelp();
    }
}
```

**handleLogin(provider_name):**
1. Call `ProviderOAuth.getConfigForProvider(provider_name)` — if null, print "Provider '{s}' does not support OAuth. Use `crushcode connect` to add an API key instead." and return.
2. Create `ProviderOAuth.init(allocator, config)`
3. Print "Opening browser for {s} login..." 
4. Call `provider_oauth.authenticate()` — this starts local callback server, prints URL, waits for callback
5. On success: print "Successfully authenticated with {s}! Tokens saved."
6. On error: print "Authentication failed: {any}"
7. Offer to set as default provider (reuse pattern from connect.zig)

**handleStatus():**
1. Load API key credentials via `Auth.init() + load()` — show providers with API keys
2. Load OAuth tokens from `~/.crushcode/provider_tokens.json` (iterate file) — show providers with OAuth
3. For each provider show: name, auth type (API key / OAuth), token expiry (if OAuth)
4. Format as a table:
```
Provider    Auth Type    Status
openrouter  OAuth        ✓ (expires in 45min)
anthropic   API Key      ✓
ollama      None         ✗
```

**handleLogout(provider_name):**
1. Call `ProviderOAuth.revokeTokens()` to remove from provider_tokens.json
2. Print "Logged out from {s}"
