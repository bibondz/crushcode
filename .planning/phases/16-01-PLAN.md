# Plan 16-01: Hashline Edit Validation

**Phase:** 16
**Goal:** Content-hash-based file edit validation preventing stale-line errors
**Depends on:** Phase 3 (AI File Operations)
**Requirements:** HASH-01, HASH-02, HASH-03, HASH-04, HASH-05

---

## Overview

Current state: File edits use line numbers or string matching. If a file changes between when AI read it and when it writes edits, the edit may apply to wrong lines — corrupting the file silently.

Target state: Every file line gets a content hash (hashline). Edit operations reference lines by hash. If the file changed externally, hashes mismatch and the edit is rejected with a clear conflict message — similar to optimistic locking in databases.

**Reference pattern:**
- **oh-my-openagent**: `hashline-edit` tool with `LINE#ID` format — content hash validation preventing stale-line errors. Each line gets a short hash derived from its content. Edits reference lines by hash. Mismatches trigger conflict detection.

## Implementation Tasks

### Task 1: Hashline Format (`src/edit/hashline.zig`)

Core hash generation:

```zig
pub const Hashline = struct {
    line_number: u32,
    content_hash: u32,  // FNV-1a hash of line content (trimmed)

    /// Generate hash for a line of content
    pub fn hash(content: []const u8) u32 {
        // Use FNV-1a 32-bit hash — fast, good distribution, no deps
        const FNV_OFFSET = 2166136261;
        const FNV_PRIME = 16777619;
        var h: u32 = FNV_OFFSET;
        for (content) |byte| {
            h ^= @as(u32, byte);
            h *%= FNV_PRIME;
        }
        return h;
    }

    /// Format hashline as "LINE#HASH" string
    pub fn format(self: Hashline, allocator) ![]const u8 {
        return std.fmt.allocPrint(allocator, "{d}#{x:0>8}", .{
            self.line_number,
            self.content_hash,
        });
    }

    /// Parse "LINE#HASH" string
    pub fn parse(str: []const u8) !Hashline;

    /// Validate that a hashline matches actual file content
    pub fn validate(self: Hashline, actual_content: []const u8) bool;
};
```

Design decisions:
- **FNV-1a 32-bit**: Fast, zero dependencies, good collision resistance for line content. 4 billion values, ~0 collision risk for typical files
- **8-char hex display**: `42#a3b4c5d6` — short, readable, copy-paste friendly
- **Trimmed content**: Hash excludes leading/trailing whitespace (editor normalization)

### Task 2: Hash Index (`src/edit/hash_index.zig`)

Per-file hash index:

```zig
pub const HashIndex = struct {
    allocator: std.mem.Allocator,
    entries: std.ArrayList(HashlineEntry),
    file_path: []const u8,
    file_hash: u32,  // Hash of entire file content for quick staleness check

    pub const HashlineEntry = struct {
        line_number: u32,
        content_hash: u32,
        original_content: []const u8,  // Non-owned reference
    };

    pub fn build(allocator, file_path) !HashIndex;
    pub fn buildFromContent(allocator, file_path, content: []const u8) !HashIndex;
    pub fn validateLine(self: *HashIndex, line_number: u32, expected_hash: u32) !ValidationResult;
    pub fn getHash(self: *HashIndex, line_number: u32) ?u32;
    pub fn isStale(self: *HashIndex, current_file_content: []const u8) bool;
    pub fn deinit(self: *HashIndex) void;
};

pub const ValidationResult = union(enum) {
    valid: void,
    stale_line: StaleLineInfo,
    line_not_found: void,
};

pub const StaleLineInfo = struct {
    line_number: u32,
    expected_hash: u32,
    actual_hash: u32,
    expected_preview: []const u8,
    actual_preview: []const u8,
};
```

Caching:
- Cache `HashIndex` in `std.StringHashMap(HashIndex)` keyed by file path
- Invalidation: compare `file_hash` (FNV-1a of entire file) before use
- Auto-rebuild if stale
- LRU eviction when cache exceeds configurable size (default: 100 files)

### Task 3: Conflict Detection (`src/edit/conflict.zig`)

```zig
pub const ConflictResolver = struct {
    allocator: std.mem.Allocator,

    pub const Conflict = struct {
        line_number: u32,
        old_content: []const u8,
        new_content: []const u8,
        edit_content: []const u8,
    };

    pub const Resolution = union(enum) {
        reject: []const u8,           // Error message
        apply: void,                   // Safe to apply
        three_way_merge: []const u8,   // Merged result
    };

    /// Detect conflicts between edit and current file state
    pub fn detectConflicts(
        self: *ConflictResolver,
        index: *HashIndex,
        edits: []const EditOperation,
    ) ![]Conflict;

    /// Resolve a conflict based on strategy
    pub fn resolve(
        self: *ConflictResolver,
        conflict: Conflict,
        strategy: ResolutionStrategy,
    ) !Resolution;

    pub const ResolutionStrategy = enum {
        reject,      // Fail with error
        force,       // Apply anyway (user override)
        merge,       // Attempt 3-way merge
    };
};
```

### Task 4: Validated Edit Operations (`src/edit/validated_edit.zig`)

```zig
pub const ValidatedEdit = struct {
    allocator: std.mem.Allocator,
    hash_cache: HashCache,  // Global hash cache across files

    pub const EditOperation = struct {
        file_path: []const u8,
        line_number: u32,
        expected_hash: u32,  // Hash of line to replace (0 = append)
        old_content: []const u8,
        new_content: []const u8,
    };

    pub const EditResult = union(enum) {
        success: void,
        conflict: ConflictInfo,
        file_not_found: void,
        hash_mismatch: HashMismatchInfo,
    };

    /// Apply edit with hash validation
    pub fn applyEdit(self: *ValidatedEdit, op: EditOperation) !EditResult;

    /// Apply multiple edits atomically (all-or-nothing)
    pub fn applyEditsAtomic(self: *ValidatedEdit, ops: []const EditOperation) ![]EditResult;

    /// Apply edit without validation (fallback for non-hashline mode)
    pub fn applyEditUnsafe(self: *ValidatedEdit, op: EditOperation) !void;

    pub fn deinit(self: *ValidatedEdit) void;
};
```

## Constraints

- Zig stdlib only — FNV-1a from `std.hash` or manual implementation
- Hash size: 32-bit (FNV-1a) — sufficient for line-level uniqueness
- No external diff/merge tools — conflict resolution is basic (reject/force)
- Memory: hash index for a 10K line file ≈ 160KB (acceptable)
- Performance: hashing 10K lines < 1ms
