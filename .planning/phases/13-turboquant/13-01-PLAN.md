### Step 2: Bit-Packing and Compression

Implement bit-packing utilities and core compression algorithms.

**Files to create:**
- `src/quantization/bitpack.zig` - Bit-packing utilities
- `src/quantization/polar_quant.zig` - PolarQuant algorithm (TurboQuantMSE)
- `src/quantization/turbo_quant.zig` - Full TurboQuant with QJL

**Changes:**
- Bit-packing: 4 values/byte (2-bit), 2 values/byte (4-bit)
- PolarQuant: Random rotation + scalar quantization
- TurboQuant: (b-1) bit PolarQuant + 1-bit QJL residual correction
- Asymmetric K/V support (preserve K, compress V)

**Verification:**
- Bit-packing round-trip lossless
- Compression ratios match expected (3.8x-6.4x)
- Inner product bias < 0.1%

## Integration Strategy

### Progressive Adoption:
1. **Phase 13.1**: Basic quantization (PolarQuant) for values only
2. **Phase 13.2**: Full TurboQuant for keys + values
3. **Phase 13.3**: Sparse V dequantization for speed
4. **Phase 13.4**: Layer-aware and temporal optimizations

### Compatibility:
- **Local LLMs**: Primary benefit for memory-constrained local inference
- **Cloud providers**: Optional optimization for cost reduction
- **Hybrid**: Use compression for long context, exact for short

### Configuration:
```toml
[quantization]
mode = "turbo4"  # turbo2, turbo3, turbo4, asymmetric, off
boundary_layers = 2  # Protect first/last N layers
recent_tokens = 1024  # Keep exact for recent context
sparse_threshold = 0.01  # Skip V dequant below this attention
```
