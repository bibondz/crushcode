# Plan 24-01: Rich System Prompt + AGENTS.md + Project Config

**Phase:** 24 (System Prompt Engineering + Project Config)
**Plan:** 24-01
**Status:** Ready for execution
**Depends on:** Phase 22 (token-aware system prompt)

---

## Goal

Replace the generic "You are a helpful AI coding assistant" system prompt with a rich, project-aware prompt that:
1. Loads project-specific instructions from `AGENTS.md` and `.crushcode/instructions.md`
2. Auto-detects project language, framework, and build system
3. Enhances the base prompt with coding best practices and tool usage guidelines
4. Injects project-type-specific context (Zig projects get `zig build` tips, etc.)

---

## Tasks

### Task 1: Create project detection module
**File:** `src/config/project.zig` (new ~150 lines)

Detect project type by looking for characteristic files:
- `build.zig` → Zig project
- `package.json` → Node.js project
- `Cargo.toml` → Rust project
- `go.mod` → Go project
- `pyproject.toml` / `requirements.txt` → Python project
- `Makefile` / `CMakeLists.txt` → C/C++ project

Return a struct:
```zig
pub const ProjectInfo = struct {
    language: []const u8,        // "zig", "javascript", "rust", etc.
    build_system: []const u8,    // "zig-build", "npm", "cargo", etc.
    framework: ?[]const u8,      // "react", "next.js", etc. (detected from deps)
    test_command: []const u8,    // "zig build test", "npm test", etc.
    build_command: []const u8,   // "zig build", "npm run build", etc.
    tips: []const u8,            // Project-type-specific coding tips
};
```

### Task 2: Load AGENTS.md and instructions
**File:** `src/config/project.zig` (same file)

Add functions:
- `loadAgentsMd(allocator) !?[]const u8` — reads `AGENTS.md` from cwd
- `loadInstructionsMd(allocator) !?[]const u8` — reads `.crushcode/instructions.md`
- Both return null if file doesn't exist (non-fatal)

### Task 3: Enhance base system prompt
**File:** `src/tui/chat_tui_app.zig`

Replace the default prompt with a richer one:
```
You are Crushcode, an expert AI coding assistant with access to the user's codebase.

## Guidelines
- Read files before editing to understand context
- Make minimal, focused changes — don't refactor unless asked
- Use the edit tool for surgical changes, write_file only for new files
- Verify your changes by reading the file after editing
- Follow existing code patterns and conventions in the project
- When running commands, prefer non-destructive operations first

## Project: {language} ({build_system})
Build: {build_command}
Test: {test_command}
{tips}
```

### Task 4: Wire into refreshEffectiveSystemPrompt
**File:** `src/tui/chat_tui_app.zig`

In `refreshEffectiveSystemPrompt`, add:
1. Load project info (one-time, cache it)
2. Load AGENTS.md content
3. Load .crushcode/instructions.md content
4. Include all in system prompt before the codebase context section

Structure:
```
[Base prompt with guidelines]
[Project info: language, build, test]
[AGENTS.md content if exists]
[.crushcode/instructions.md if exists]
## Codebase Context
{context}
## Available Tools
{tools}
```

### Task 5: Build wiring
**File:** `build.zig`

Add `project.zig` as a module and wire it to `tui_mod`.
