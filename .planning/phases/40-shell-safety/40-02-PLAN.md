    1. Add `timed_out: bool = false` field to existing `ProcessResult` struct (default false preserves backward compatibility).

    2. Create `runShellCommandWithTimeout`:
    ```zig
    /// Execute a shell command with optional timeout in seconds.
    /// When timeout_seconds is null or 0, behaves like runShellCommand (no timeout).
    /// When timeout triggers, child is killed (SIGKILL on POSIX), returns exit_code=124
    /// and timed_out=true with any output collected so far.
    /// Output is capped at max_output_bytes (default 1MB) per stream.
    pub fn runShellCommandWithTimeout(
        allocator: std.mem.Allocator,
        command: []const u8,
        timeout_seconds: ?u32,
    ) !ProcessResult
    ```

    3. Implementation:
       - Spawn child with `stdout_behavior = .Pipe`, `stderr_behavior = .Pipe`
       - If no timeout (null or 0): call `child.collectOutput()` then `child.wait()` (existing proven path from runShellCommand)
       - If timeout: use the threading pattern from shell.zig's `waitWithTimeout()` but WITH pipe reading:
         a. Create `SharedState` struct with `stdout_buf`, `stderr_buf` (ArrayListUnmanaged), `term`, `done`, `lock`
         b. Spawn reader thread: reads `child.stdout` pipe into `stdout_buf`, reads `child.stderr` into `stderr_buf`, then calls `child.wait()` to get term. Max read = `1024 * 1024` bytes per stream. On pipe close (read returns 0), stop reading.
         c. Main thread: poll `state.done` every 10ms. If elapsed > timeout_ms and not done → kill child:
            - POSIX: `posix.kill(child.id, posix.SIG.KILL)` (SIGKILL for immediate termination)
            - Windows: skip kill (same limitation as existing code — future enhancement)
            - Set `state.timed_out = true`
         d. Join reader thread
         e. Extract stdout/stderr from shared state, compute exit_code

    4. Keep existing `runShellCommand()` unchanged (backward compat). Add doc note: "For timeout support, use runShellCommandWithTimeout()".

    5. Keep all existing tests passing.
