    **Phase 2 (GREEN):** Implement:

    ```zig
    const std = @import("std");
    const Allocator = std.mem.Allocator;

    /// Sandbox enforcement mode.
    pub const SandboxMode = enum {
        off,    // No restrictions
        cwd,    // Restrict writes to current working directory (default)
        custom, // Restrict writes to a user-specified directory
    };

    pub const SandboxConfig = struct {
        mode: SandboxMode = .cwd,
        custom_path: ?[]const u8 = null,
    };

    pub const SandboxResult = struct {
        allowed: bool,
        reason: ?[]const u8 = null,  // If blocked, human-readable reason
    };

    pub const SandboxChecker = struct {
        allocator: Allocator,
        config: SandboxConfig,
        cwd: []const u8,

        pub fn init(allocator: Allocator, config: SandboxConfig, cwd: []const u8) SandboxChecker;
        pub fn deinit(self: *SandboxChecker) void;

        /// Check if a shell command is allowed under the current sandbox rules.
        /// Returns allowed=true if no write restrictions are violated.
        pub fn isCommandAllowed(self: *SandboxChecker, command: []const u8) SandboxResult;
    };
    ```

    **Write detection logic** (`extractWriteTargets` internal function):
    1. Scan command string for these write-like patterns (case-insensitive substring match):
       - Shell redirects: `>`, `>>` followed by a path
       - Commands that write: `tee `, `cp `, `mv `, `install `, `dd `, `rsync `, `scp `
       - Pattern: `command_word [args] target_path` — extract the LAST argument as target for cp/mv/install
    2. For redirects (`>`, `>>`): extract the path immediately following the redirect operator
    3. For write commands: extract the last non-flag argument as the target path
    4. Resolve each target path to absolute (prepend cwd if relative)
    5. Compare: does resolved path start with allowed_prefix (cwd or custom_path)?

    **isCommandAllowed flow:**
    1. If mode == .off → return allowed=true
    2. Extract write targets from command
    3. If no write targets found → return allowed=true (read-only is fine)
    4. For each write target: resolve to absolute path, check starts-with allowed_prefix
    5. If any target fails → return allowed=false with reason like "Write to /tmp/out.txt is outside sandbox (/home/user)"

    **Phase 3 (REFACTOR):** Clean up string matching. All 10 tests must pass.
