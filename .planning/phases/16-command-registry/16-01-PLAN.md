  <action>
Create `src/cli/registry.zig` — a comptime command registry that replaces the if-else dispatch.

**Design:**

1. **Normalize all handlers to one signature.** The main dispatch block has 3 handler signatures. Normalize to:
```zig
pub const HandlerFn = *const fn (args_mod.Args, *config_mod.Config) anyerror!void;
```
Handlers that don't need config just ignore the second parameter (`_ = config;`). Create thin wrappers for `printHelp()` and `printVersion()` that match this signature.

2. **Command struct:**
```zig
pub const Command = struct {
    name: []const u8,
    handler: HandlerFn,
    description: []const u8,
    aliases: []const []const u8,  // e.g. help has aliases --help, -h
};
```

3. **Comptime registration.** Use `std.StaticStringMap` (available in Zig 0.13+) to build a compile-time hash map from command names to Command structs. This gives O(1) lookup at runtime.

4. **dispatch function:**
```zig
pub fn dispatch(command: []const u8, args: args_mod.Args, config: *config_mod.Config) !void
```
- Look up command in the comptime map
- If found → call handler
- If not found → return `error.CommandNotFound`
- Special handling: aliases (help/--help/-h, version/--version/-v) map to same handler

5. **contains function:**
```zig
pub fn contains(command: []const u8) bool
```
- Returns true if command is in the registry (including aliases)
- Replaces `isCommandRecognized()`

6. **helpText function:**
```zig
pub fn helpText() []const u8
```
- Returns a comptime-formatted help string derived from registered commands
- Each command line: `  {name:<15}{description}`
- This replaces the hardcoded help string in handlers.zig `printHelp()`

8. **Wrapper functions for handlers with wrong signatures.** Create internal wrappers:
```zig
fn wrapHelp(args: args_mod.Args, _: *config_mod.Config) !void {
    _ = args;
    try handlers.printHelp();
}
fn wrapVersion(args: args_mod.Args, _: *config_mod.Config) !void {
    _ = args;
    try handlers.printVersion();
}
fn wrapArgsOnly(comptime handler: fn(args_mod.Args) anyerror!void) HandlerFn {
    return struct {
        fn call(args: args_mod.Args, _: *config_mod.Config) anyerror!void {
            return handler(args);
        }
    }.call;
}
```

2. **Delete `isCommandRecognized()` function entirely** (lines 45-84, ~40 lines). Replaced by `registry.contains()`.

3. **Replace the entire dispatch block** (lines 201-280, ~80 lines) with:
```zig
const main_result: anyerror!void = blk: {
    if (registry.dispatch(parsed_args.command, parsed_args, &config)) |_| {
        break :blk;
    } else |err| switch (err) {
        error.CommandNotFound => {
            // Not a built-in command — try plugin fallback
            if (try commands.tryHandlePluginCommand(parsed_args.command)) {
                break :blk;
            }
            err_print("Error: Unknown command '{s}'\n\n", .{parsed_args.command});
            try commands.printHelp();
            break :blk error.UnknownCommand;
        },
        else => break :blk err,
    }
    break :blk;
};
```

This is ~15 lines instead of 80+40 = 120 lines. The dispatch becomes: lookup in comptime map → call handler → if not found try plugin → if not plugin show help.

4. **Keep the no-command default** (lines 157-185) — this is the TUI launch logic, not part of the dispatch chain. Leave untouched.

5. **Keep everything else** — the SIGINT handler, config loading, BrokenPipe handling. Only remove isCommandRecognized and the dispatch block.

6. **Also update the unknown command check** (lines 188-196):
Replace:
```zig
if (!isCommandRecognized(parsed_args.command)) {
```
With:
```zig
if (!registry.contains(parsed_args.command)) {
```

Actually, since `dispatch()` handles CommandNotFound internally, we can REMOVE this pre-check entirely. The dispatch function handles unknown commands. But we need to keep the plugin fallback. The flow becomes:

```
No command → launch TUI (unchanged)
Has command → registry.dispatch()
  - Found → call handler
  - Not found → try plugin fallback → if no plugin, show help + error
```

So the entire main() after "No command" block simplifies to just the dispatch call.

**Keep handlers.zig mostly unchanged** — it's already a thin re-export layer. The only thing that COULD change is printHelp(), but since we're wrapping it in the registry, we can leave it for now and improve later. The help text in handlers.zig stays as-is; the registry's helpText() is an enhancement for the future.
