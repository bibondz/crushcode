# Phase 38: Streaming Diff Preview + Hunk Apply/Reject

**Goal:** Interactive diff preview where users see each edit as a colored hunk and can approve/reject individual changes before they're applied.

**Milestone:** v0.37.0
**Status:** Planning

---

## Problem

AI edits files directly with no preview. The permission dialog shows a read-only diff, but users can only accept/reject the ENTIRE change. No way to approve some edits while rejecting others within the same file.

**Current flow:** AI → edit tool → permission dialog (all-or-nothing) → apply/reject entire file
**Desired flow:** AI → edit tool → diff preview (per-hunk) → [y/n/a/q] per hunk → apply selected hunks only

pub const DiffLineKind = enum { equal, insert, delete };

    kind: DiffLineKind,

## Implementation Plan

### Task 1: DiffPreviewWidget (`src/tui/widgets/diff_preview.zig`)

**New file.** Interactive diff preview widget that shows hunks one at a time with apply/reject controls.

**Widget Layout:**
```
┌─── Edit Preview: src/main.zig ────────────────────┐
│ Hunk 2 of 5  │  +12 -3 lines                       │
│────────────────────────────────────────────────────│
│ 45 | pub fn main() !void {                         │
│ 46 |     const args = try parseArgs();             │
│    | -   std.debug.print("hello", .{});            │
│    | +   const stdout = std.io.getStdOut();        │
│    | +   try stdout.writeAll("hello\n");           │
│ 47 |     try run(args);                            │
│────────────────────────────────────────────────────│
│ [y] Apply  [n] Reject  [a] Apply All  [q] Quit    │
│ Applied: 1  Rejected: 0  Remaining: 3              │
└────────────────────────────────────────────────────┘
```

**State:**
```zig
pub const DiffPreviewWidget = struct {
    hunks: []const myers.DiffHunk,
    current_hunk: usize,
    hunk_decisions: []HunkDecision,  // .pending, .applied, .rejected
    file_path: []const u8,
    theme: *const theme_mod.Theme,
    total_adds: u32,
    total_dels: u32,
};

pub const HunkDecision = enum { pending, applied, rejected };
```

**Methods:**
- `draw()` — render current hunk with context lines, stats, navigation
- `handleKey()` — process [y/n/a/q/j/k] keys
- `currentDecision()` — get current hunk decision
- `allDecided()` — check if all hunks have been decided
- `appliedHunks()` — return slice of applied hunks
- `computeResultingText()` — apply only approved hunks to original text

**Key bindings:**
- `y` — apply current hunk, move to next
- `n` — reject current hunk, move to next
- `a` — apply all remaining hunks
- `q` — reject all remaining hunks
- `j` / `↓` — next hunk (just view, don't decide)
- `k` / `↑` — previous hunk
- `?` — show help

### Task 2: Integrate into Tool Execution Flow

**Modify `src/tui/chat_tui_app.zig`:**

Add new message type for diff preview:
```zig
pub const Msg = union(enum) {
    // ... existing variants ...
    diff_preview_response: struct {
        tool_call_id: []const u8,
        tool_name: []const u8,
        file_path: []const u8,
        hunks: []myers.DiffHunk,
        original_content: []const u8,
        new_content: []const u8,
        hunk_decisions: []HunkDecision,
    },
};
```

Modify `executeToolCalls()` flow:
1. For edit/write_file tools, instead of going straight to permission:
   - Call `previewEditDiff()` / `previewWriteDiff()` to get `DiffResult`
   - If 0 hunks (no change), skip preview, auto-approve
   - If 1 hunk, show simplified preview (single hunk, y/n)
   - If 2+ hunks, show full `DiffPreviewWidget`
2. On hunk decisions complete:
   - Compute resulting text with only applied hunks
   - Execute tool with modified content (only applied changes)
   - Add tool result message to chat

### Task 3: Hunk-Level Apply Logic

**Modify `src/chat/tool_executors.zig`:**

Add `applySelectedHunks()`:
```zig
/// Apply only the selected hunks to the original text.
/// Returns owned slice of the resulting text.
pub fn applySelectedHunks(
    allocator: std.mem.Allocator,
    original_text: []const u8,
    hunks: []const myers.DiffHunk,
    decisions: []const HunkDecision,
) ![]const u8
```

Logic:
1. Split original text into lines
2. For each hunk:
   - If `.applied`: replace old lines with new lines
   - If `.rejected`: keep original lines unchanged
3. Join lines back into text

6. Rejected hunks leave original text unchanged
7. Stats show applied/rejected/remaining counts
8. Build passes clean
