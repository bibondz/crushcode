# Plan 15-01: Token Usage Tracking

**Phase:** 15
**Goal:** Comprehensive token and cost tracking for AI interactions with budget alerts
**Depends on:** Phase 14 (Streaming — usage data extracted from streaming responses)
**Requirements:** TOKEN-01, TOKEN-02, TOKEN-03, TOKEN-04, TOKEN-05

---

## Overview

Current state: `ChatResponse.Usage` has `prompt_tokens`, `completion_tokens`, `total_tokens` — basic fields, not used anywhere. No cost tracking, no session accumulation, no budget alerts.

Target state: Every AI request records detailed token usage (input, output, cache_read, cache_write). Sessions accumulate totals. Cost estimation per provider. `crushcode usage` command shows reports. Budget alerts fire when approaching limits.

**Reference patterns:**
- **multica**: `TokenUsage` struct with `InputTokens`, `OutputTokens`, `CacheReadTokens`, `CacheWriteTokens` per model, tracked in `Session`
- **oh-my-openagent**: `ModelCapabilitiesCache` for provider metadata, token tracking in streaming responses

## Implementation Tasks

### Task 1: Enhanced TokenUsage (`src/usage/tracker.zig`)

Core tracking structure:

```zig
pub const UsageTracker = struct {
    allocator: std.mem.Allocator,
    data_dir: []const u8,  // ~/.crushcode/usage/

    // In-memory session tracking
    session_usage: SessionUsage,

    // Cumulative tracking (persisted)
    daily_usage: DailyUsage,
    total_usage: CumulativeUsage,

    pub const SessionUsage = struct {
        request_count: u32,
        input_tokens: u64,
        output_tokens: u64,
        cache_read_tokens: u64,
        cache_write_tokens: u64,
        total_tokens: u64,
        estimated_cost_usd: f64,
        by_provider: std.StringHashMap(ProviderUsage),
    };

    pub const ProviderUsage = struct {
        provider: []const u8,
        model: []const u8,
        request_count: u32,
        input_tokens: u64,
        output_tokens: u64,
        cache_read_tokens: u64,
        cache_write_tokens: u64,
        cost_usd: f64,
    };

    pub fn init(allocator, data_dir) !UsageTracker;
    pub fn recordUsage(self: *UsageTracker, provider, model, usage: TokenUsage) !void;
    pub fn getSessionUsage(self: *UsageTracker) SessionUsage;
    pub fn getDailyUsage(self: *UsageTracker) !DailyUsage;
    pub fn getCumulativeUsage(self: *UsageTracker) !CumulativeUsage;
    pub fn persist(self: *UsageTracker) !void;
    pub fn resetSession(self: *UsageTracker) void;
    pub fn deinit(self: *UsageTracker) void;
};
```

Persistence:
- Session usage in `~/.crushcode/usage/session_<timestamp>.json`
- Daily usage accumulated in `~/.crushcode/usage/daily_<YYYY-MM-DD>.json`
- Auto-trim files older than 30 days

### Task 2: Pricing Tables (`src/usage/pricing.zig`)

Provider pricing data:

```zig
pub const ModelPricing = struct {
    provider: []const u8,
    model: []const u8,
    input_price_per_1m: f64,   // USD per 1M input tokens
    output_price_per_1m: f64,  // USD per 1M output tokens
    cache_read_price_per_1m: ?f64,  // Cache read discount (if applicable)
    cache_write_price_per_1m: ?f64, // Cache write cost (if applicable)
};

pub const PricingTable = struct {
    entries: std.StringHashMap(ModelPricing),

    pub fn init(allocator) !PricingTable;
    pub fn getPrice(self: *PricingTable, provider, model) ?ModelPricing;
    pub fn estimateCost(self: *PricingTable, provider, model, usage: TokenUsage) f64;
};
```

Include pricing for:

- Anthropic (Claude 3.5 Sonnet, Haiku, Opus)

- OpenRouter (passthrough pricing)

### Task 3: Budget Manager (`src/usage/budget.zig`)

Budget enforcement:

```zig
pub const BudgetConfig = struct {
    daily_limit_usd: ?f64,
    monthly_limit_usd: ?f64,
    per_session_limit_usd: ?f64,
    alert_threshold_pct: f64 = 0.8,  // Alert at 80%
};

pub const BudgetManager = struct {
    config: BudgetConfig,
    tracker: *UsageTracker,

    pub fn checkBudget(self: *BudgetManager) !BudgetStatus;
    pub fn shouldAlert(self: *BudgetManager) bool;
    pub fn isOverBudget(self: *BudgetManager) bool;

    pub const BudgetStatus = struct {
        daily_spent: f64,
        daily_limit: ?f64,
        monthly_spent: f64,
        monthly_limit: ?f64,
        session_spent: f64,
        session_limit: ?f64,
        percent_used: f64,
    };
};
```

Behavior:
- `shouldAlert()` returns true when usage > threshold% of any limit
- `isOverBudget()` blocks new requests (or warns)
- Alert printed to stderr: `⚠ Budget alert: ${X.XX} spent ({Y}% of ${Z.ZZ} daily limit)`

## Constraints

- Zig stdlib only
- Pricing data hardcoded (can be updated via config in future)
- JSON persistence via `std.json.stringify`
- No network calls for pricing data — all local
- `f64` for cost calculations (sufficient precision for USD)
