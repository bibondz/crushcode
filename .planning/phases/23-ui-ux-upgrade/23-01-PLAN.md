**Types to define:**
```zig
pub const DiffLineKind = enum { equal, insert, delete };
pub const DiffLine = struct { kind: DiffLineKind, content: []const u8, old_line_num: ?u32, new_line_num: ?u32 };
pub const DiffHunk = struct { old_start: u32, old_count: u32, new_start: u32, new_count: u32, lines: []DiffLine };
pub const DiffResult = struct { hunks: []DiffHunk, allocator: Allocator, pub fn deinit(self: *DiffResult) void };
```

**Algorithm implementation (Myers O(ND)):**
1. Split old_text and new_text into line arrays
2. Run Myers algorithm:
   - `V` array for furthest-reaching D-path endpoints on each diagonal k
   - For each edit distance D from 0 to N+M:
     - For each diagonal k from -D to D step 2:
       - Choose between extending down (delete) or right (insert)
       - Snake: extend along diagonal (equal lines)
       - Track trace for backtracking
   - If reached (N, M), backtrack through trace to get edit script
3. Convert edit script to DiffLine array with line numbers
4. Group into hunks: collect contiguous changes with 3 lines of context before/after
5. Assign correct old/new line numbers to each line

**Additional functions:**
- `pub fn diff(allocator, old_text: []const u8, new_text: []const u8) !DiffResult` — main entry point
- `pub fn formatUnifiedDiff(allocator, result: *DiffResult, old_path: []const u8, new_path: []const u8) ![]const u8` — produces unified diff text like:
  ```
  --- a/file.zig
  +++ b/file.zig
  @@ -10,5 +10,6 @@
   context line
  -removed line
  +added line
  ```
- `pub fn diffToEditScript(allocator, old_text: []const u8, new_text: []const u8) ![]DiffLine` — flat list without hunk grouping (useful for tool_executors preview)

**Implementation notes:**
- Use `std.ArrayList` for dynamic arrays throughout
- Allocator as first parameter to all public functions
- `defer` for all allocations
- The V array can use `std.ArrayList(isize)` with index offset for negative k values (k ranges from -D to D, so index = k + D)
- Trace should be `std.ArrayList(std.ArrayList(isize))` — one V snapshot per D iteration
- For backtracking: start at (N, M), walk backwards through trace to recover edit script
- Context lines for hunks: 3 lines before and after each change group; merge hunks if context overlaps
- No external imports needed — pure Zig stdlib (std.mem, std.ArrayList)

**Do NOT:**
- Import anything beyond `std` (no vaxis, no file_compat — this is a pure algorithm module)
- Use `std.process` — no I/O in this module
- Make the API async or callback-based — simple synchronous functions

```zig
const myers = @import("myers");

pub fn showInlineDiff(self: *DiffVisualizer, old_text: []const u8, new_text: []const u8) !void {
    // Use myers.diff() → format as colored inline diff to stdout
    var result = try myers.MyersDiff.diff(self.allocator, old_text, new_text);
    defer result.deinit();
    // Print each hunk with colors (red for -, green for +, dim for context)
    const stdout = file_compat.File.stdout().writer();
    for (result.hunks) |hunk| {
        for (hunk.lines) |line| {
            switch (line.kind) {
                .delete => stdout.print("- {s}\n", .{line.content}) catch {},
                .insert => stdout.print("+ {s}\n", .{line.content}) catch {},
                .equal => stdout.print("  {s}\n", .{line.content}) catch {},
            }
        }
    }
}

pub fn showUnifiedDiff(self: *DiffVisualizer, old_path: []const u8, old_text: []const u8, new_text: []const u8) !void {
    // Use myers to produce unified diff, then print to stdout
    var result = try myers.MyersDiff.diff(self.allocator, old_text, new_text);
    defer result.deinit();
    const unified = try myers.formatUnifiedDiff(self.allocator, &result, old_path, old_path);
    defer self.allocator.free(unified);
    const stdout = file_compat.File.stdout().writer();
    stdout.print("{s}", .{unified}) catch {};
}
```

Keep `compareFiles` as-is (reads files, calls `showUnifiedDiff`).
