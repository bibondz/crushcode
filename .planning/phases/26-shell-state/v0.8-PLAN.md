## Phase 26 — Shell State Persistence

### 26-01: Shell State Manager
**Create:** `src/shell/state.zig`
- `ShellState` struct tracking: `cwd: []const u8`, `env: std.StringHashMap([]const u8)`, `last_exit_code: u8`
- `init(allocator, initial_cwd)` — captures current process cwd
- `updateCwd(new_path)` — resolves relative to current cwd, validates existence
- `setEnv(key, value)` / `getEnv(key)` — persistent env vars
- `buildEnvPairs(allocator)` — builds `[?][:0]const u8` for `std.process.Child`
- `deinit()` — cleanup
- **Integration:** Used by `executeShellCommand` to pass cwd + env to child processes

### 26-02: Background Job Manager
**Extend:** `src/commands/jobs.zig` (currently stubbed)
- `runBackground(command, cwd, env)` — spawns `std.process.Child` with `spawn_async`
- Track PID, start_time, status (running/done/failed/killed)
- `getOutput(job_id)` — reads stdout/stderr from temp files
- `listJobs()` — returns all jobs with status
- `killJob(job_id)` — sends SIGTERM
- `cleanup(max_age)` — auto-cleanup jobs older than max_age
- **Constants:** max 50 concurrent, 8hr auto-cleanup (from crush reference)

### 26-03: Shell History
**Create:** `src/shell/history.zig`
- `ShellHistory` with append-only log at `~/.crushcode/history`
- `add(command, exit_code, cwd, timestamp)`
- `search(query, limit)` — prefix search
- `recent(limit)` — last N commands
- **Format:** One JSON line per entry: `{"cmd":"...","exit":0,"cwd":"...","ts":123456}`

### 26-04: Wire Shell State into Agent Loop + TUI
- Agent loop's shell tool uses `ShellState` for cwd/env
- TUI shows current cwd in status bar
- `/cd <path>` command to change cwd
- `/jobs` command to list background jobs
- `/history` command to show recent commands

## Phase 27 — Command Security

### 27-01: Global Command Blocklist
**Create:** `src/shell/blocklist.zig`
- `CommandBlocklist` with two-tier blocking:
  - `CommandsBlocker`: exact match (curl, sudo, rm, shutdown, reboot, mkfs)
  - `ArgumentsBlocker`: subcommand+flag combos (rm -rf, git push --force, chmod 777)
- `isBlocked(command)` — returns blocked status + reason
- `loadFromConfig(config_dir)` — loads from `~/.crushcode/blocklist.json`
- `addBlock(command_or_pattern)` — runtime addition
- **Default blocklist** embedded in code (from crush reference)

### 27-02: Safe Commands Whitelist
**Create:** `src/shell/safelist.zig`
- `SafeCommandList` with pre-approved read-only commands:
  - git: log, status, diff, show, branch, remote
  - ls, pwd, echo, cat, head, tail, wc, sort, uniq
  - find, grep, rg, fd (read-only flags only)
  - ps, top, df, du, free, uptime, hostname, whoami
- `isSafe(command)` — returns true if command is pre-approved (skip permission prompt)
- `loadFromConfig(config_dir)` — loads user additions from `~/.crushcode/safelist.json`

### 27-03: Output Truncation
**Extend:** `src/commands/shell.zig`
- Outputs >30K chars truncated by splitting at midpoint (crush pattern)
- `... [N lines truncated] ...` notice
- Exit code and interruption status always included
- `<cwd>` tags for directory tracking

### 27-04: Wire Blocklist into Permission Flow
- Before executing any shell command:
  1. Check `CommandBlocklist.isBlocked()` → deny with reason
  2. Check `SafeCommandList.isSafe()` → auto-approve
  3. Fall through to existing `PermissionEvaluator` for everything else
- Config files auto-created on first run with defaults

## Phase 28 — Permission Persistence

### 28-01: Persistent Permission Config
**Extend:** `src/permission/types.zig`
- `PermissionConfig.save(allocator, path)` — serialize to JSON
- `PermissionConfig.load(allocator, path)` — deserialize from JSON
- `addSessionGrant(tool, path_pattern, action)` — remember approval for session
- `isSessionGranted(tool, path_pattern)` — check session grant
- Auto-save on grant, auto-load on startup from `~/.crushcode/permissions.json`

### 28-02: Permission Audit Log
**Create:** `src/permission/audit.zig`
- `PermissionAuditLogger` — append-only log at `~/.crushcode/audit.jsonl`
- `log(tool, action, pattern, granted, timestamp)`
- `recent(limit)` — last N decisions
- `/permissions log` command to view recent decisions

### 28-03: --yolo Mode
**Extend:** `src/cli/args.zig` + `src/commands/chat.zig`
- `--yolo` flag → sets `PermissionMode.bypassPermissions`
- `--auto` flag → sets `PermissionMode.auto` (auto-allow safe, ask on risky)
- `--plan` flag → sets `PermissionMode.plan` (read-only)
- Document in `--help` output

## Phase 29 — Non-Interactive Mode

### 29-01: Piped Stdin Support
**Create:** `src/commands/run.zig`
- `crushcode run "prompt"` — single prompt, output to stdout
- `cat file | crushcode run "summarize"` — piped stdin as context
- `crushcode run "generate" > output.md` — stdout redirection
- Detect piped stdin via `std.posix.isatty(std.posix.STDIN_FILENO)`
- If stdin is piped, read all stdin and prepend to prompt

### 29-02: Session Continuation
**Extend:** `src/cli/args.zig` + `src/commands/chat.zig`
- `--continue` flag → load last session and continue
- `--session <id>` flag → load specific session by ID or hash
- `--model <provider/model>` flag → override model for this run
- Session ID printed on startup for reference

### 29-03: Batch Mode
**Create:** `src/commands/batch.zig`
- `crushcode batch prompts.txt` — read prompts from file, one per line
- Each prompt gets a response, output to stdout
- `--output-dir <dir>` — save responses as separate files
- `--stop-on-error` — halt on first error

### 29-04: CLI Completion + Help
**Extend:** `src/cli/args.zig`
- `crushcode --help` — comprehensive help with examples
- `crushcode list --providers` — list available providers
- `crushcode list --models <provider>` — list models for provider
- `crushcode config --show` — show current configuration
- `crushcode config --set key=value` — set config value
