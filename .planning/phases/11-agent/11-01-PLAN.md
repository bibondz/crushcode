  <name>Task 1: Create LSPManager module</name>
  <files>src/tui/lsp_manager.zig</files>
  <action>
Create `src/tui/lsp_manager.zig` — a high-level wrapper around `LSPClient` that manages multiple LSP server instances (one per language) for the TUI context.

The module must contain:

1. **FileDiagnostics struct:**
   - `file_path: []const u8` (owned)
   - `uri: []const u8` (owned)
   - `errors: u32`
   - `warnings: u32`
   - `infos: u32`
   - `diagnostics: []lsp.LSPClient.Diagnostic` (owned copies)

2. **LanguageServer struct** — wraps one LSPClient per language:
   - `client: lsp.LSPClient`
   - `language: []const u8` (owned)
   - `started: bool`

3. **LSPManager struct:**
   - `allocator: std.mem.Allocator`
   - `servers: std.ArrayList(LanguageServer)` — one per language
   - `file_diagnostics: std.ArrayList(FileDiagnostics)` — cached per file
   - `workspace_uri: []const u8` (owned)
   - `enabled: bool` — can be toggled off

4. **Key methods on LSPManager:**
   - `init(allocator) LSPManager` — initialize empty
   - `deinit(self)` — shutdown all servers, free all owned data
   - `onFileOpened(self, file_path: []const u8) !void` — auto-detect language from extension (copy the extension-to-language mapping from lsp_handler.zig's `detectLSPLanguage`), start LSP server for that language if not already running (using `lsp.getLSPServer`), open the document, read file contents with `std.fs.cwd().readFileAlloc`, call `client.openDocument()`. Skip if `enabled` is false or language detection returns null or getLSPServer returns error (log and continue, don't fail the whole operation).
   - `refreshDiagnostics(self) !void` — for each file in file_diagnostics, poll `client.getDiagnostics(uri)` and update the FileDiagnostics entry. Count errors/warnings/infos by severity. Must drain notifications first (getDiagnostics already calls drainNotifications internally).
   - `getDiagnosticsSummary(self) ![]FileDiagnostics` — return snapshot of current diagnostics
   - `totalErrors(self) u32` — sum of errors across all files
   - `totalWarnings(self) u32` — sum of warnings across all files
   - `setWorkspaceUri(self, workspace_path: []const u8) !void` — convert path to file URI, store for LSP initialize

5. **Helper functions (copy pattern from lsp_handler.zig):**
   - `detectLanguage(file_path: []const u8) ?[]const u8` — same extension mapping (.zig→zig, .rs→rust, .go→go, .ts/.tsx→typescript, .js/.jsx/.mjs/.cjs→javascript, .py→python, .java→java)
   - `pathToFileUri(allocator, file_path) ![]const u8` — resolve to absolute, convert backslashes, prepend "file://"
   - `readFileContent(allocator, file_path) ![]const u8` — wrapper around readFileAlloc with 10MB limit

6. **Memory safety:**
   - All strings owned by LSPManager (file paths, URIs) must be freed in deinit
   - Use `errdefer` for allocations in init-style functions
   - FileDiagnostics.diagnostics must be deep-copied from LSPClient (messages are owned strings) — use the pattern from LSPClient.cloneDiagnostics

7. **Error resilience:** LSP operations should NEVER crash the TUI. All errors from LSPClient (server not found, connection failed, timeout) should be caught with `catch |err| { std.log.warn("LSP: {}", .{err}); return; }` pattern. The TUI must continue working even if no LSP server is installed.

Do NOT import from lsp_handler.zig — it's a CLI command handler with no public exports. Reimplement the small helper functions (detectLanguage, pathToFileUri) directly in lsp_manager.zig.

Import `lsp` module (which maps to src/lsp/client.zig per build.zig line 276). Also import `array_list_compat` and `file_compat` (universally available per build.zig lines 357-359).
