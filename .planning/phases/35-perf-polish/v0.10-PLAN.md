# v0.10.0 — Performance + Polish

**Milestone:** v0.10.0
**Theme:** Production hardening — output management, tool validation, git workflow polish
**Date:** 2026-04-17

---

## Goals

1. Prevent token waste from oversized tool outputs (output truncation)
2. Make the tool ecosystem safer with runtime input validation (tool schemas)
3. Expose git worktree isolation to users and improve diff workflows

---

## Phase Breakdown

### Phase 35: Smart Output Truncation
**Effort:** Medium (wiring existing code + enhancements)

**Current State:**
- `src/commands/shell.zig` has `truncateOutputAlloc` (30KB) but it's **never called**
- `executeShellTool` passes up to 1MB raw output to AI via `collectOutput(allocator, &stdout, &stderr, 1024 * 1024)`
- No truncation for grep, glob, or read_file outputs either

**Implementation:**
1. Wire `truncateOutputAlloc` into `executeShellTool` in `tool_executors.zig`
2. Add truncation to `executeGrepTool` and `executeGlobTool` results
3. Make truncation threshold configurable (default 30KB, override in config)
4. Use "smart truncation" — split at midpoint, show head + tail with `[N lines truncated]` notice (crush pattern)
5. Include truncation notice in tool result so AI knows output was shortened

**Files:**
- `src/chat/tool_executors.zig` — Wire truncation into all tool executors
- `src/commands/shell.zig` — Enhance `truncateOutputAlloc` to be midpoint-aware
- `src/config/config.zig` — Add `max_tool_output_bytes` config option

**Success Criteria:**
- Shell outputs >30KB are truncated with clear notice
- AI receives `[1234 lines truncated]` message and can ask for more
- Config option allows users to change the threshold

---

### Phase 36: Tool Input Validation
**Effort:** Medium (build on existing schema infrastructure)

**Current State:**
- `ToolSchema` has `parameters: []const u8` (raw JSON Schema string)
- Input is parsed via `std.json.parseFromSlice` into Zig structs only
- No runtime JSON Schema validation (min/max, regex, enum constraints)
- `src/config/default_tools.json` has schemas but they're just forwarded to providers

**Implementation:**
1. Add a `validateToolInput` function that checks required fields exist before execution
2. Validate file paths: reject empty paths, warn on absolute paths outside cwd
3. Validate command strings: reject empty commands, flag obviously malformed commands
4. Add per-tool argument validation in each `execute*Tool` function (guard clauses)
5. Return structured validation errors that the AI can understand and self-correct

**Files:**
- `src/chat/tool_executors.zig` — Add validation guards to each tool executor
- `src/protocol/tool_types.zig` — Add `validateInput` method to ToolSchema
- `src/config/tool_loader.zig` — Parse schema required fields at load time

**Success Criteria:**
- Empty/missing required fields return clear error to AI
- Invalid file paths are caught before filesystem operations
- AI gets actionable error messages it can self-correct from

---

### Phase 37: Git Workflow Polish
**Effort:** Low-Medium (CLI exposure of existing infrastructure)

**Current State:**
- `WorktreeManager` in `src/agent/worktree.zig` has full lifecycle (create/remove/cleanup/listActive)
- `GitSkill` in `src/commands/git.zig` has 30+ git operations but no `worktree` subcommand
- `DiffVisualizer` in `src/diff/visualizer.zig` produces git-style unified diffs
- Worktree isolation is used internally by agents but not exposed to users

**Implementation:**
1. Add `crushcode worktree` CLI command with subcommands: create, list, remove, cleanup
2. Wire to existing `WorktreeManager` (just CLI exposure, no new logic)
3. Add `/worktree` slash command in interactive chat for quick isolation
4. Improve `crushcode git diff` to use `DiffVisualizer` for prettier output
5. Add `--worktree` flag to `crushcode run` for isolated execution

**Files:**
- `src/cli/registry.zig` — Register worktree command
- `src/commands/handlers.zig` — Add `handleWorktree`
- `src/commands/chat.zig` — Add `/worktree` slash command handler
- `src/commands/git.zig` — Enhance diff output with DiffVisualizer

**Success Criteria:**
- `crushcode worktree create/list/remove` works from CLI
- `/worktree` in interactive chat creates isolated environment
- Git diff output uses DiffVisualizer for color-coded unified format

## Risk Assessment

| Risk | Mitigation |
|------|-----------|
| Truncation hides important output | Configurable threshold + AI can request full output |
| Validation blocks valid inputs | Only validate clearly invalid cases; permissive defaults |
| Worktree commands fail on non-git repos | Graceful error message with setup instructions |

---

## Metrics

- Token savings from output truncation (estimate: 30-50% on large outputs)
- Reduction in "tool failed" errors from input validation
- User adoption of worktree isolation
