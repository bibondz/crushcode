
**`generateHtmlReport(allocator, run: *const TraceRun, config: HtmlReportConfig) ![]const u8`**

Builds a complete HTML string (allocated with allocator) containing:

1. **`<html><head>`** — UTF-8 charset, viewport meta, title
2. **`<style>` block** (inline CSS, ~100 lines):
   - Dark theme with CSS variables: `--bg: #1a1a2e`, `--surface: #16213e`, `--accent: #0f3460`, `--text: #e6e6e6`, `--error: #e74c3c`, `--ok: #2ecc71`, `--timeout: #f39c12`
   - Body: font-family monospace, max-width 1200px, margin auto
   - Summary cards: CSS grid 4-column layout
   - Timeline bar: horizontal bars with width proportional to latency
   - Waterfall chart: stacked vertical bars, each span a row with name, duration bar, timing
   - Span tree: indented list with expand/collapse via inline JS
   - Table styling for cost/token breakdown
3. **Summary section** — 4 cards:
   - Total Duration: `{total_duration_ms}ms`
   - Total Cost: `${total_cost_usd:.6}` (use `{d:.6}` in Zig fmt)
   - Span Count: `{span_count}` (with breakdown: `{llm_count} LLM, {tool_count} Tool, {agent_count} Agent`)
   - Errors: `{error_count} errors, {timeout_count} timeouts`
4. **Timeline section** — horizontal bar for each span:
   - Calculate min_start_ns across all spans
   - Each span gets a horizontal bar: left offset = `(span.start_time_ns - min_start_ns) / total_range_ns * 100%`, width = `span.latency_ms / total_duration_ms * 100%`
   - Color by kind: llm=blue, tool=green, agent=purple, chain=orange, guardrail=red
   - Status indicator: ok=solid, error=striped, timeout=dashed border
5. **Waterfall chart** — vertical stacked view:
   - Each span as a row: `| name | duration bar | latency_ms |`
   - Bar colored by kind, with tooltip showing model/tokens
   - Child spans indented under parents (build parent→children map from parent_hex)
6. **Span tree** — hierarchical `<ul>` with inline JS toggle:
   - Root spans (no parent) at top level
   - Children nested under parents
   - Each span shows: kind icon, name, status badge, latency, model
   - Click to expand/collapse children
   - JS function: `function toggleSpan(id) { ... }` — inline in `<script>` tag
7. **Cost/Token breakdown** — `<table>`:
   - One row per LLM span: model, prompt_tokens, completion_tokens, total_tokens, cost_usd
   - Footer row with totals
8. **Failure diagnosis section** (only if errors/timeouts exist):
   - Table: span_name, failure_class, cause_hint, status_message
9. **Closing `</body></html>`**

Implementation approach:
- Use `std.ArrayList(u8)` as a buffer, write HTML chunks via `writer().print()`
- Build parent→children map: iterate spans, create a `std.StringHashMap(ArrayList(*ParsedSpan))` keyed by span_id_hex
- Root spans = spans where parent_hex is null
- For waterfall: sort spans by start_time_ns
- Use `std.fmt.allocPrint` for small strings, direct writer.print for HTML chunks

IMPORTANT: All CSS and JS must be INLINE (no external files, no CDN links). The HTML must be fully self-contained.
IMPORTANT: Use `{d:.6}` NOT `{d:.6f}` for Zig 0.15.2 float formatting.
IMPORTANT: Escape HTML entities in span names and status messages: `<` → `&lt;`, `>` → `&gt;`, `&` → `&amp;`.
