**`pub fn handleTrace(args: args_mod.Args) !void`**

Parse `args.remaining` for subcommand and flags:

**Subcommand: `list`** — `crushcode trace list [--kind K] [--status S]`
- Get traces_dir from `env_mod.getTracesDir(allocator)` (or fallback: `~/.crushcode/traces/`)
- Call `listTraceFiles(allocator, traces_dir)` to get file list
- For each file, extract trace_id from filename (strip `.jsonl`)
- Call `loadTrace` for each, print a table row:
  ```
  {trace_id_short}  {span_count} spans  {total_duration_ms}ms  ${total_cost_usd:.6}  {error_count} err  {first_ts}
  ```
  (trace_id_short = first 8 chars of hex)
- If `--kind` or `--status` provided, load each trace, filter, show counts

**Subcommand: `show`** — `crushcode trace show <run-id> [--kind K] [--status S]`
- run-id can be full hex (32 chars) or short (8+ chars, prefix match against files)
- Load trace via `loadTrace(allocator, traces_dir, run_id_hex)`
- Print summary:
  ```
  Trace: {trace_id_hex}
  Duration: {total_duration_ms}ms
  Cost: ${total_cost_usd:.6}
  Spans: {span_count} ({llm_count} LLM, {tool_count} Tool, {agent_count} Agent)
  Errors: {error_count} errors, {timeout_count} timeouts
  ```
- Print span table:
  ```
  #  Name                 Kind   Status  Latency    Model        Tokens   Cost
  1  chat_completion      llm    ok      1234ms     gpt-4        150      $0.001
  2  execute_tool         tool   ok      567ms      -            -        -
  ```
- If filters applied, only show matching spans
- If errors exist, print failure diagnosis:
  ```
  Failures:
    ⚠ chat_completion: timeout — Request exceeded timeout limit
  ```

**Subcommand: `export`** — `crushcode trace export <run-id> --format html|json|md [--output PATH]`
- Load trace
- Based on `--format` flag:
  - `html` → call `generateHtmlReport(allocator, run, .{})`
  - `json` → call `exportJson(allocator, run)`
  - `md` → call `exportMarkdown(allocator, run)`
- Default output path: `./trace-{trace_id_short}.{html|json|md}`
- If `--output PATH` specified, use that path
- Write result to file via `std.fs.cwd().createFile(path, .{})`
- Print: `Exported trace to {path} ({size} bytes)`

**Subcommand: `compare`** — `crushcode trace compare <run-a> <run-b>`
- Load both traces (resolve short IDs same as show)
- Call `compareTraces(allocator, run_a, run_b)`
- Print comparison:
  ```
  Run Comparison: {run_a_id} vs {run_b_id}

  Metric            Run A        Run B        Delta
  Total Cost        $0.001       $0.002       +100%
  Latency           1234ms       567ms        -54%
  Span Count        5            3            -40%
  LLM Calls         3            2            -33%
  Tool Calls        2            1            -50%
  Errors            0            1            +1

  Tool Usage:
    execute_tool     2            1            -1

  Verdict: Run A is better (6/8 metrics favor Run A)
  ```

**Helper: resolveTraceId(traces_dir, input_id) ![]const u8**
- If input_id is 32 chars → return as-is
- If shorter → list trace files, find one whose filename starts with input_id
- Return the full hex trace_id (caller frees)

**Helper: getTracesDir(allocator) ![]const u8**
- Check `getEnv("CRUSHCODE_TRACES_DIR")` first
- Fallback to `~/.crushcode/traces/`
- Use env_mod.getHomeDir or similar existing pattern

Use `file_compat.File.stdout().writer().print()` for all output.
Use `color_mod` for colored output: green for ok, red for errors, yellow for warnings.

IMPORTANT: The import names must match what build.zig registers. The trace modules will be imported as `trace_reader`, `trace_html_report`, `trace_json_export`, `trace_markdown_export`, `trace_comparison` (configured in build.zig in Task 2).
