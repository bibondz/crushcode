**MetricDelta** — a single comparison dimension:
```zig
pub const MetricDelta = struct {
    name: []const u8,           // static string, not owned (e.g., "Total Cost", "Latency", "Tool Count")
    run_a_value: f64,
    run_b_value: f64,
    delta: f64,                 // run_b - run_a (positive = run_b is higher)
    delta_percent: f64,         // percentage change: (delta / run_a_value) * 100, 0 if run_a is 0
    lower_is_better: bool,      // true for cost/latency/errors, false for throughput
};
```

**Verdict** — overall comparison result:
```zig
pub const Verdict = enum {
    run_a_better,
    run_b_better,
    similar,
};

pub const ToolUsageDiff = struct {
    tool_name: []const u8,      // owned
    run_a_count: usize,
    run_b_count: usize,
    delta: i32,                 // run_b - run_a
};
```

**TraceComparison** — full comparison result:
```zig
pub const TraceComparison = struct {
    run_a_id: []const u8,       // owned
    run_b_id: []const u8,       // owned
    metrics: []MetricDelta,     // owned slice
    tool_diffs: []ToolUsageDiff, // owned slice
    verdict: Verdict,
    summary: []const u8,        // owned — human-readable summary string
    allocator: Allocator,

    pub fn deinit(self: *TraceComparison) void { ... }
};
```

**`compareTraces(allocator, run_a: *const TraceRun, run_b: *const TraceRun) !*TraceComparison`**

Compute these MetricDeltas:
1. **Total Cost**: `total_cost_usd` — lower is better
2. **Total Latency**: `total_duration_ms` — lower is better
3. **Span Count**: `span_count` — lower is better (fewer steps = more efficient)
4. **LLM Calls**: `llm_count` — lower is better
5. **Tool Calls**: `tool_count` — lower is better
6. **Error Count**: `error_count` — lower is better
7. **Token Efficiency**: `total_tokens / span_count` (average tokens per span) — lower is better
8. **Cost Efficiency**: `total_cost_usd / total_tokens` (cost per token) — lower is better (compute total tokens by summing all span total_tokens)

For ToolUsageDiff:
- Build a `std.StringHashMap(usize)` for each run, counting tool spans by name
- Compare: for each unique tool name, compute delta

For Verdict:
- Count how many metrics favor run_a vs run_b (delta < -5% for lower-is-better = run_a, delta > 5% = run_b, else neutral)
- If run_a wins > 50% of metrics → run_a_better
- If run_b wins > 50% → run_b_better
- Else → similar

Build summary string:
```
"Run A ({id}) vs Run B ({id})
 Cost: {a_cost} → {b_cost} ({delta}%)
 Latency: {a_lat}ms → {b_lat}ms ({delta}%)
 Errors: {a_err} → {b_err}
 Verdict: {verdict}"
```

IMPORTANT: Use `{d:.3}` NOT `{d:.3f}` for float formatting.
IMPORTANT: `array_list_compat.ArrayList` for all dynamic lists.
