**Key functions:**

1. `parseSpanRecord(allocator, json_line: []const u8) !*ParsedSpan`
   - Parse JSON line using `std.json.parseFromSlice` with an anonymous struct matching the JSONL format
   - Map string kind ("llm"/"tool"/"agent"/"chain"/"guardrail") to SpanKind enum via `std.meta.stringToEnum`
   - Map string status ("ok"/"error"/"timeout") to SpanStatus enum
   - Parse ISO8601 ts field back to epoch nanoseconds (inverse of writer.zig epochToIso8601)
   - Compute end_time_ns from start_time_ns + latency_ms * 1_000_000 if latency_ms present
   - All string fields are duped with allocator

2. `listTraceFiles(allocator, traces_dir: []const u8) ![][]const u8`
   - Read directory entries, filter to `*.jsonl` files
   - Return sorted array of full paths (sorted by filename = sorted by trace_id hex)
   - Use `std.fs.cwd().openDir(traces_dir, .{ .iterate = true })` + `iterate()`

3. `loadTrace(allocator, traces_dir: []const u8, trace_id_hex: []const u8) !*TraceRun`
   - Open `{traces_dir}/{trace_id_hex}.jsonl`
   - Read all lines, parse each with parseSpanRecord
   - Compute TraceRun metadata: total_cost_usd (sum), total_duration_ms (max latency), counts by kind/status
   - Return populated TraceRun

4. `filterSpans(allocator, run: *const TraceRun, filter: SpanFilter) ![]*ParsedSpan`
   - Iterate run.spans, apply filter.matches(), collect matches
   - Return slice (allocated via allocator)

5. `classifyFailure(span: *const ParsedSpan) FailureDiagnosis`
   - If span.status == .timeout → FailureClass.timeout, hint "Request exceeded timeout limit"
   - If status_message contains "rate" (case-insensitive) → .rate_limit, hint "Provider rate limit exceeded"
   - If status_message contains "auth" or "401" or "403" → .auth_error, hint "Authentication failure"
   - If status_message contains "network" or "connect" or "ECONNREFUSED" → .network_error, hint "Network connectivity issue"
   - If status_message contains "500" or "502" or "503" → .server_error, hint "Provider server error"
   - If span.kind == .tool and status == .error → .tool_error, hint "Tool execution failed"
   - Else → .unknown, hint "Unrecognized failure pattern"

6. `diagnoseFailures(allocator, run: *const TraceRun) ![]FailureDiagnosis`
   - Filter error/timeout spans, classify each, return array

7. `iso8601ToEpochNs(iso: []const u8) !i64`
   - Parse "YYYY-MM-DDTHH:MM:SSZ" format back to epoch seconds, then ×1_000_000_000 for ns
   - Use std.fmt.parseInt for each component

Use `std.json.parseFromSlice` with a struct matching the JSONL format (same shape as writer.zig SpanRecord but with field names matching the JSON keys). For Zig 0.15.2: use `std.json.parseFromSlice` with `.allocate = .alloc_always` for string fields.

IMPORTANT: Zig 0.15.2 format specifier is `{d:.3}` NOT `{d:.3f}`. Use `std.fmt.parseFloat` for cost_usd.
IMPORTANT: `array_list_compat.ArrayList` is the compat ArrayList used throughout the project, NOT `std.ArrayList`.
