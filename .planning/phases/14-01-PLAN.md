# Plan 14-01: Streaming Session Abstraction

**Phase:** 14
**Goal:** Unified streaming session interface across all AI providers with real-time token display
**Depends on:** Phase 1 (Core Infrastructure), Phase 11 (Agent Framework)
**Requirements:** STREAM-01, STREAM-02, STREAM-03, STREAM-04, STREAM-05

---

## Overview

Current state: `AIClient.sendChat()` buffers the entire HTTP response before parsing and returning. Users see nothing until the full response arrives. Ollama has basic NDJSON chunk accumulation but no real-time display.

Target state: A `StreamingSession` abstraction that wraps all providers behind a unified streaming interface, displaying tokens in real-time as they arrive, with provider-specific parsers for NDJSON, SSE, and JSON-RPC formats.

**Reference patterns:**
- **multica**: `Backend` interface with streaming `Session` channels, provider-specific parsers (Claude `stream-json`, Codex JSON-RPC, OpenCode NDJSON)
- **oh-my-openagent**: `event-stream-processor.ts` for handling message deltas and completion events with error recovery

## Implementation Tasks

### Task 1: Stream Types (`src/streaming/types.zig`)

Define core streaming types:

```zig
pub const StreamEvent = union(enum) {
    token: []const u8,          // Single token received
    tool_call: ToolCallStart,   // Tool invocation begins
    tool_result: ToolCallResult,// Tool execution result
    thinking: []const u8,       // Thinking/reasoning token (if supported)
    done: StreamDone,           // Stream completed
    error: StreamError,         // Stream error
};

pub const StreamDone = struct {
    finish_reason: ?[]const u8,
    usage: ?TokenUsage,
};

pub const TokenUsage = struct {
    input_tokens: u32,
    output_tokens: u32,
    cache_read_tokens: u32,
    cache_write_tokens: u32,
};

pub const ToolCallStart = struct {
    id: []const u8,
    name: []const u8,
    arguments: []const u8,  // Partial JSON, accumulated over chunks
};

pub const ToolCallResult = struct {
    id: []const u8,
    output: []const u8,
};

pub const StreamError = struct {
    code: u32,
    message: []const u8,
};

pub const StreamOptions = struct {
    on_event: ?*const fn (StreamEvent) void = null,
    buffer_size: usize = 4096,
    display_tokens: bool = true,
    timeout_ms: u64 = 30000,
};
```

### Task 2: NDJSON Parser (`src/streaming/parsers/ndjson.zig`)

Parse newline-delimited JSON from Ollama and similar providers:

- Read HTTP response in chunks
- Split by `\n` boundaries
- Parse each complete JSON line
- Emit `StreamEvent.token` for each content chunk
- Emit `StreamEvent.done` when `done: true`

### Task 3: SSE Parser (`src/streaming/parsers/sse.zig`)

Parse Server-Sent Events from OpenAI/Anthropic/OpenRouter:

- Parse `data: ` prefixed lines
- Handle `data: [DONE]` termination
- Extract delta content from `choices[0].delta.content`
- Parse tool call deltas from `choices[0].delta.tool_calls`
- Handle Anthropic-specific `content_block_delta` format
- Handle `usage` field in final chunk

### Task 4: JSON-RPC Parser (`src/streaming/parsers/jsonrpc.zig`)

Parse JSON-RPC 2.0 notifications for Codex-style providers:

- Parse `{"jsonrpc":"2.0","method":"...", "params":{...}}` messages
- Extract content from `params.delta` or `params.content`
- Map method names to event types

### Task 5: StreamingSession (`src/streaming/session.zig`)

Unified session interface:

```zig
pub const StreamingSession = struct {
    allocator: std.mem.Allocator,
    provider: []const u8,
    model: []const u8,
    state: SessionState,
    response_buffer: ResponseBuffer,
    token_count: u32,

    pub const SessionState = enum {
        pending, streaming, paused, completed, cancelled, failed,
    };

    pub fn init(allocator, provider, model, options) !StreamingSession;
    pub fn start(self: *StreamingSession, client: *std.http.Client, request) !void;
    pub fn pause(self: *StreamingSession) void;
    pub fn resume(self: *StreamingSession) !void;
    pub fn cancel(self: *StreamingSession) void;
    pub fn waitForCompletion(self: *StreamingSession) !StreamResult;
    pub fn getFullResponse(self: *StreamingSession) ?[]const u8;
    pub fn deinit(self: *StreamingSession) void;
};
```

Key behavior:
- `start()` opens HTTP connection with `stream: true` in request body
- Reads response body in chunks via `std.http.Client`
- Routes chunks to provider-specific parser
- Emits `StreamEvent`s via callback or polling
- `pause()` stops reading from response body (buffers incoming)
- `cancel()` closes the connection
- `waitForCompletion()` blocks until stream ends, returns final `StreamResult`

## Constraints

- Zig stdlib only — no external dependencies
- Must work with `std.http.Client` for HTTP streaming (response body reader)
- All parsing via `std.json`
- Terminal output via `std.io.getStdOut()` / ANSI escape codes
- Memory-safe with proper `deinit()` on all allocs
